The whole story in 4 steps

  1. 1 The Podcast publishes a new episode. GitHub Actions wakes up Saturday morning and grabs it. (One-way arrow: GitHub only pulls; nothing is sent back to the podcast.)
  2. 2 GitHub Actions sends the audio + notes to Groq's AI, which sends the summary back. (Two-way arrow: data goes up, summary comes down.) The result is saved in the Repo and Pages publishes it.
  3. 3 A visitor types a question into the chatbox. The question travels to the Cloudflare Worker and the answer travels back. (Two-way arrow.)
  4. 4 Cloudflare relays the question to Groq's AI, gets the answer, and sends it back. (Two-way arrow.) Your secret password never leaves Cloudflare.

Reading the arrows: An arrow with one head (→) means data flows that way only. An arrow with two heads (↔) means a request goes one way and a response comes back, like asking a question and getting an answer.
